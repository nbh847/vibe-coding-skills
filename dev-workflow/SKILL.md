# Dev Workflow

## 通用规则

- 使用中文与用户交互
- 注意模块的复用，功能要区分清楚
- 不要一股脑全写在一个文件里，按照不同用途分类保存
- 尽量不要写重复的代码，能复用就复用
- 区域层级功能块需要设计的好一点，让项目结构层次分明
- 无论是前端还是后端，都要遵循良好的代码组织原则

## 后端项目

- 使用 venv 虚拟环境执行所有脚本
- 不安装额外依赖库，直接使用已安装的库

## 前端项目

- CSS 样式分类整理到合适的路径
- 自动识别项目结构，找到或创建合适的样式存放位置
- 仅整理当前正在编辑的页面的 CSS 样式
- 不修改已存在页面的样式（避免大规模重构）
- 功能开发完成后必须自检，确认没有错误

## Git 提交与推送

- 执行 git push 命令前必须停止并让用户确认
- 展示即将推送的提交信息和变更文件列表
- 等待用户明确确认后再执行推送操作
